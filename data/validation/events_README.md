# data/validation/events_README.md

このファイルは、events.csv の入力ルールを定義する。
events.csv は、cases.csv に登録済みの案件について、続報・経過（イベント）を
時系列で記録するためのファイルである。

全般ルール（unknown の使用、CSVのクォート規則など）は
[README.md](./README.md) および [CLAUDE.md](../../CLAUDE.md) に従う。

## 基本方針

- 1行 = 1イベント。同一案件の続報を確認した場合は、cases.csv に新規行を
  追加せず、events.csv に新しいイベント行を追加する。
- 別製品・別目的の施策は、同一自治体であっても別案件（別 case_id）として
  cases.csv に新規登録し、そのイベントも別 case_id に紐づけて記録する。
- 単なる名称変更や通常の続報だけでは、別案件を作らない。
- イベントを追加したら、対応する cases.csv の current_status を
  最新のイベントに合わせて更新する。
- 出典に明記されていない事実を推測して埋めない。
- 不明な項目は空欄にせず、`unknown` を入力する。「未確認」という
  日本語トークンはCSV内では使用しない。
- human_reviewed が `true` になるまで、そのイベントは下書き（未確定）として扱う。

## 列定義

| 列名 | 内容 | 入力ルール |
|---|---|---|
| event_id | イベントを一意に識別するID | `MGE-000001`, `MGE-000002`... の形式に統一する（正規表現 `^MGE-[0-9]{6}$`）。一度発行したIDは変更・再利用しない。 |
| case_id | 対象案件のID | cases.csv に存在する `MGR-0001` 形式のIDを使用する。存在しないIDは使用しない。 |
| event_date | イベントの発生・公表日 | `YYYY-MM-DD` 形式。日付が特定できない場合は `unknown`。 |
| event_type | イベントの種別 | 下記「イベント種別一覧」から1つを選択する。 |
| event_title | イベントを簡潔に表す名称 | 例：「実証結果公表」「本導入決定」のように、内容が分かる簡潔な文言にする。 |
| event_summary | イベントの内容 | 公式情報から確認できる事実を簡潔に記載する。推測は記載しない。不明な場合は `unknown`。 |
| organization | イベントを発表・実施した主体 | 自治体・企業名を入力する。複数ある場合は半角セミコロン `;` で区切って列挙する（例：`A社;B社`）。不明な場合は `unknown`。 |
| source_url | イベントの根拠となるURL | 必須。可能な限り一次情報（自治体・企業の公式発表）のURLを入力する。 |
| source_type | source_url の情報源種別 | `自治体公式` / `企業公式` / `官公庁` / `議会資料` / `予算・入札` / `報道` / `その他` のいずれかを入力する。 |
| last_checked | 最終確認日 | このイベントの内容を最後に確認・更新した日付（`YYYY-MM-DD`）。 |
| human_reviewed | 人間によるレビュー済みか | `true` / `false` / `unknown`。初回登録時は `false` とする。`false` の間は下書き（未確定情報）として扱う。 |
| notes | 補足事項 | 判断上の注意点、他案件・他イベントとの関係など、他の列に書ききれない事実を自由記述する。推測は書かず、不明な場合は `unknown`。 |

## イベント種別一覧（event_type）

- 構想発表
- 連携協定
- 公募開始
- 採択・契約
- 実証開始
- 実証終了
- 実証結果公表
- 本導入
- 共同利用開始
- 効果公表
- 他自治体展開
- サービス終了
- 続報確認
- その他

## 入力時の注意

- すべてのイベントに出典URL（source_url）を付ける。
- 元記事の本文や画像はそのまま転載せず、事実の要約と出典URLの記載にとどめる。
- カンマ `,` を含む値を入力する場合は、CSVの規則に従いダブルクォートで囲む。
